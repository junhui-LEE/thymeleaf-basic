1. 표준 HTML 주석

1. 표준 HTML 주석:
*자바스크립트의 표준 HTML 주석은 타임리프가 렌더링 하지 않고, 그대로 남겨둔다.*
최종적으로 랜더링 된 결과를 보면(소스보기로 보면) th가 있는
<span th:text="${data}">html data</span>가 주석으로 처리 된 다음에
그대로 보여지는 것을 확인할 수 있다.
-타임리프는 최종적인 HTML파일로의 변환은 하지 않고 따라서 타임리프 측에서 주석처리 되어 th:*가 있는 코드 그대로 남겨두기 때문에
최종적인 HTML에서는 타임리프 측에서 주석처리 되어 th:*가 있는 코드를 그대로 볼 수 있다.

2. 타임리프 파서 주석

2. 타임리프 파서 주석
타임리프 파서 주석은 타임리프의 진짜 주석이다. 렌더링에서 주석 부분을 제거한다.
타임리프 파서 주석은 <!--/* ... */--> 이렇게 사용하고 이렇게 사용하면
타임리프에서 HTML로의 변환 자체를 막아버리기 때문에 타임리프 기능으로서
역할을 하는 [[${data}]]는 아얘 HTML로의 변환 자체가 안되기 때문에 최종적인
HTML소스를 소스보기 해서 보면 아얘 코드 자체가 없는 것을 확인할 수 있다.
<!--/* [[${data}]] */--> 라는 코드도 찾아 볼 수 없다. , 다시말해
타임리프 파서 차원에서 그냥 주석처리를 꽉 해서 아얘 출력인 안되도록 하는 것이다.
<!--/*-->
<span th:text="${data}">html data</span>
<!--*/--> 이것도 가능하다.

3. 타임리프 프로토타입 주석

3. 타임리프 프로토타입 주석 타임리프 프로토타입은 약간 특이한데, HTML 주석에 약간의 구문을 더했다. HTML 파일을 웹 브라우저에서 그대로 열어보면 HTML 주석이기 때문에 이 부분이 웹 브라우저가 렌더링 하지 않는다.(페이지 소스보기 하면 "타임리프 프로토타입 주석"은 코드로 보인다.) 타임리프 렌더링을 거치면 이 부분이 정상 렌더링 된다. *쉽게 이야기해서 HTML 파일을 그대로 열어보면 주석처리가 되지만, 타임리프를 렌더링 한 경우에만 보이는 기능이다.*
